NOL & NOLE Specification

NOL (Notation Object Language) is a high-performance, strictly typed configuration language designed for security and predictability. NOLE extends it with controlled evaluation features.

Secure by Default

Strict UTF-8 validation, no null bytes, and enforced depth/size limits.

Fast & Deterministic

Predictable parsing with no side effects or hidden behaviors.

NOL Core Specification

1. Encoding & Validation

All NOL files MUST be valid UTF-8. Parsers must reject:

  • Null bytes (raw or escaped)
  • UTF-16 surrogate pairs
  • Overlong encodings

Normalization: All keys are normalized to Unicode NFC for comparison.

2. Syntax & Types

NOL supports a rich set of static types:

Scalars

name: "John Doe"      # String
age: 30               # Integer
pi: 3.14159           # Float
active: true          # Boolean
meta: null            # Null

Sections & Nesting

[server]
host: "127.0.0.1"
port: 8080

[server.tls]
enabled: true

NOLE Extended Features

NOLE adds evaluated capabilities while maintaining strict safety controls.

Evaluation Order

Evaluations MUST occur in this specific sequence:

  1. Merge Resolution (<<)
  2. Environment Variable Access
  3. Interpolation (${...})
  4. Type Coercion (<type>)

Environment Variables

Uses a Dual Allowlist model. Access is only granted if the variable is in both the application and document allowlists.

# Document level allowlist
[_env]
allowed: ["API_KEY", "DB_PORT"]

# Usage
database: {
  port: { env: "DB_PORT" }
}

Interpolation

References local values using dot-notation paths.

base_url: "api.example.com"
endpoint: "https://${base_url}/v1"

Anchors & Merges

Define templates and reuse them to reduce duplication.

base_db: &db {
  host: "localhost"
  port: 5432
}

prod_db: {
  <<: *db
  host: "db.production"
}

Multi-Language Implementations

C++17

C++ Implementation

High-performance, object-oriented API using modern C++ features.

#include "nole.hpp"

auto res = NOLE::parse(input, true, {"VAR"});
if (res.ok()) {
    auto val = res.value.at("server.port");
    std::cout << val->asInt() << std::endl;
}
C99

C Implementation

Header-only library with no dependencies beyond libc.

#include "nole.h"

char* allowed[] = {"VAR"};
nol_val* doc = nole_parse(input, true, allowed, 1);
nol_val* v = nole_at(doc, "key");
printf("%s\n", v->u.s);
nole_free(doc);
Python 3.8+

Python Implementation

Idiomatic Python implementation returning standard dicts/lists.

from nole import parse, Document

doc = parse(input_str, n=True, e=["VAR"])
print(doc['server']['port'])
Node.js

JavaScript Implementation

Works in Node.js environments with full NOLE support.

const { parse } = require('./nole.js');

const doc = parse(input, true, ['VAR']);
console.log(doc.server.port);
Rust

Rust Implementation

Safe, memory-efficient implementation using idiomatic Enums.

use nole::{Document, Value};

let doc = Document::parse(input, true, vec!["VAR".into()]).unwrap();
if let Some(Value::Int(p)) = doc.get("server.port") {
    println!("{}", p);
}
PHP 7.4+

PHP Implementation

Namespaced implementation using stdClass and standard arrays.

use Nole;

$doc = Nole\parse($input, true, ['VAR']);
echo $doc->server->port;
